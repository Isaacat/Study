git repository: 파일이나 폴더를 변경 이력 별로 구분지어 저장하는 곳

remote repository: 파일이 원격 저장소 전용 서버에서 관리되면 여러 사람이 함께 공유하기 위한 저장소.

local repository: 내 PC에 파일이 저장되는 개인 전용 저장소.

commit: 파일 및 폴더의 추가, 변경 사항을 저장소에 기록하려면 'commit'해야 하며, 시간 순으로 저장됨. 

commit 메시지 작성 양식: 
1.커밋 내의 변경 내용을 요약
2.빈 칸
3.변경한 이유

work tree: 'git'에서의 폴더.

index: 'commit'하기 전의 저장소와 작업 트리 사이에 존재하는 공간.

stage: 'index'에 파일 상태를 기록하는 행위.

push: 'local repository'의 변경 사항을 'remote repository'로 업로드하는 행위.

clone: 'remote repository'의 내용을 전부 다운로드하는 것.

pull: 'remote repository'의 변경 사항을 'local repository'로 업로드하는 행위.

branch: 여러 개발자들이 동시에 다양한 작업을 할 수 있게 만들어주는 것.
